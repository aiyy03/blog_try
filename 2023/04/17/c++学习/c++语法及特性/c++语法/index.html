c++_syntax

  1. 数组 array.size()

  2. &有2种用法：1.取值2.引用，对于引用这个用法就是同一个地址取不同的变量名字（叫不同的名字），改变引用变量，原变量也随之改变。

    *也有两种用途：1.声明2.间接引用

  3. 函数传递变量三种，值传递，地址传递，引用传递。

  4. Extern关键字不能放在函数里面，要放在全局变量的位置。确定变量可以使用其他cpp文件声明的变量，extern可以置于变量或者函数前，以表示变量或者函数的定义在别的文件中，可以这么理解，extern 是用来在另一个文件中声明一个全局变量或函数。另外，extern也可用来进行链接指定。 有点类似python中的import可以自动寻找同一工程下面同名函数。一个变量被使用，要被定义(定义的同时声明了)，extern声明并不分配空间。

  5. 指针和引用的区别：（1）指针不用初始化值，引用声明的时候需要指定绑定的变量。

    右值引用： int &&a = 1；目的是为了优化空间（减少复制次数，增加临时变量存活周期）

    常量左值引用：const int &a = 1；（缺点只能是常量）

  6. 类型别名：auto不知道返回值类型，自动声明。

  7. 不要在头文件中使用using，std是一个标准的作用域。

  8. 在c++中一般就用string 去表示字符串，等价于char *。

  9. 函数重载，相同的函数名。

    (1)函数名称必须相同。

    (2)参数列表必须不同（个数不同、类型不同、参数排列顺序不同等）。

    (3)函数的返回类型可以相同也可以不相同。

    (4)仅仅返回类型不同不足以成为函数的重载。

  10. 函数指针可以实现函数的回调，做的事情就是：在一个函数中，不改变其他执行语句，通过函数指针指向不同的函数，从而执行不同的函数。（当然也可以通过输入字符串匹配进行选择，感觉不够高效）

    (16条消息) 回调函数和普通调用函数_大龙哥。的博客-CSDN博客_回调函数和直接调用函数

  11. 对于指针来说一般指向的是栈（局部变量），通过new和malloc的方式能指向堆（这里开辟的空间不是指针变量，而是指针变量所指向的地方）。（我的理解可能是由于栈空间不够了，需要注意的是指向堆的指针，不要改变其指向的地址，不然开辟的堆无法释放，所以定义拷贝构造函数时要加const）

  12. 浅拷贝和深拷贝，（1）浅拷贝就是两个对象同时指向同一个位置 （2）深拷贝就是两个对象存储的值一致但是两个位置。

  13. 拷贝构造函数，是一种特殊的构造函数，它在创建对象时，是使用同一类中之前创建的对象来初始化新创建的对象。一般类中会自带拷贝构造函数，但这是浅拷贝。如果类带有指针变量，并有动态内存分配，则它必须有一个拷贝构造函数。 暂时不理解条件为什么要加动态内存分配，我的理解是为了深拷贝，所以要自己重新写拷贝构造函数。

  14. 拷贝赋值函数，就不是在构造的时候赋值。由于一开始存储数据的空间不一定一样，所以一开始要先删除 所指向的空间然后创建一个，再拷贝。注意在删除之前要进行自我赋值检测。

  15. **弹性结构体，成员变量写成a[]**，其实就是个指针。但是这个指针所指向的空间会自动分配为结构体变量的下一个。

    （1）空数组指长度为0的数组，仅可在类或者结构体中声明。

    定义空数组的好处有：

    1. 不占用空间，如果是指针的话还需要4字节空间，而空数组不占用任何空间

    2. 无需初始化，数组名直接就是所在的偏移，如果是指针则需要给指针赋值

      通常用来申请buff空间的写法是：

      malloc(sizeof(strTest) + bufflen);

      这样即可一次给结构体和缓冲区分配内存，减少使用和维护成本.。

  16. string和string.h是两个东西，一个c++一个是c，cstring对应string.h。

  17. 句柄handle，是一个void类型指针貌似。（待更新）

  18. 智能指针，指向动态空间，可以自动释放空间。（待更新）（需要特殊方式定义）

  19. （1）函数重载：函数名相同，变量列表必须不相同。

    （2）函数重写：（函数名和函数参数列表必须完全一样）父类虚函数，不一定在子类重写，重写之后调用父类方法依旧是子类方法。父类纯虚函数,子类一定要重写。在C++中子类对象调用父类函数可通过域操作符(::)

    （3）函数重定义：函数名相同，变量列表可以不相同，子类对象时候调子类方法，父类对象时调父类方法。

  20. 模板类的实现和定义应该尽量都放在头文件中实现。

  21. 类的组合在该类中用其他类的对象。

  22. 函数也有地址，函数名代表函数地址。对于 fun 和 &fun 应该这样理解：

    他们两值完全一样。

    (1). fun 是函数的首地址，它的类型是 void ()
    (2). &fun 表示一个指向函数 fun 这个对象的地址， 它的类型是 void (*)()
    因此 fun 和 &fun 所代表的地址值是一样的，但类型不一样。fun 是一个函数
    &fun 表达式的值是一个指针！

  23. c++提供了关键字explicit,阻止了隐式构造。

  24. const成员函数，该函数可以被 const 对象调用，并且不能修改类中的任何成员变量。

    operator有2种用法，第一个是重载操作运算符，一种是隐式类型转换。

  25. NULL在c++种宏定义为0，函数重载时难以判断到底是int 0还是 (void*) 0.

  26. RAII，全称为Resource Acquisition Is Initialization，汉语是“资源获取即初始化”。就是资源获取的时候将其封装在某类的object中。利用”栈资源会在相应object的生命周期结束时自动销毁”来自动释放资源，即，构造函数创建时初始化获取资源，并将资源释放写在析构函数中。所以这个RAII其实就是和智能指针的实现是类似的。（核心思想：将资源和类对象绑定，用构造申请，析构释放）

    （1） lock_guard 和**unique_lock**都是这一思想，在创造对象的时候lock()，对象销毁的时候unlock()。给互斥量上锁，以为着只有当前线程能进对资源进行操作。

    （2）**unique_lock**还能和条件变量一起使用，实现线程同步通信，同时也能手动解锁。

  27. 条件变量（为了实现线程同步通信）的几个关键点：

    （1）wait()函数的内部实现是：先释放了互斥量的锁，然后阻塞（已经unlock）以等待条件为真；

    （2）notify系列函数需在unlock之后再被调用；（不一定）

    （3）套路是：

    ​ a. A线程拿住锁，然后wait,此时已经释放锁，只是阻塞了在等待条件为真；
    b. B线程拿住锁，做一些业务处理，然后令条件为真，释放锁，再调用notify函数；
    ​ c. A线程被唤醒，接着运行。
